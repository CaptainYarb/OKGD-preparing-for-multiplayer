Terminology

  • Node: A physical or virtual machine on which server code is hosted.
  • Server: The code which controls the multiplayer interaction. Manages the network between players and game activity.
  • Client: The actual customer facing game. Clients connect to the server.
  • Player: A gamer which is connected to the server, using the client.

Designing Your Netcode!

  • Use a cross-platform codebase:
    • Java
    • Any C, C++
    • Python
    • Node.JS
  • Avoid Peer to Peer for competitive gameplay
  • Separate client & server code
    • Client manages controls and display
    • Server manages gameplay

Designing Your Netcode!

  • Never trust the client
    • Provide data only as needed
    • Expect Hack clients
    • Expect out of date clients
  • Utilize the best network transit layers for the job
    • UDP - Fast, but potentially unreliable
    • TCP - Rock solid, structured, potentially slow
    • REST - Great for data/stat polling

Troll Repellant

  • Network Attacks WILL happen
    • DoS / DDoS
    • Man in the Middle
  • Hack Detection
    • Client side detection almost always fails
    • Server side detection is king
  • Tracking Account Hopping
    • IP Address
    • Hardware Signature

Supporting Gaming Communties

  • Mods/Plugins APIs are a MUST
  • Effective player management
  • Simple GUI for server managment
  • File based config
    1. TXT
    2. YAML
    3. KeyValues
    4. JSON*

What Game Server Providers Want

  • IP Address/Port management
  • RCON Protocol or similar status/query protocol
  • Stateful process STDOUT
  • Consistant Config format
  • Linux Support
  • Headless (nogui, terminal) process
